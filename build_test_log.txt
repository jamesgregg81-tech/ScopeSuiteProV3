2026-04-28 09:50 local - Analyzer PNG preview canvas raise fix

Bug reported:
- Previewing screen_capture.png from an Analyze This Screen report failed with:
  wrong # args: should be ".!notebook.!frame5.!frame2.!frame.!canvas raise tagOrId ?aboveThis?"

Root cause:
- The report preview code called report_canvas.tkraise().
- For tk.Canvas this can resolve to the Canvas item-raise command instead of the generic widget raise command.
- Tk then expected a canvas tag/id and failed before rendering the PNG preview.

Fix applied:
- Changed the report image preview pane to call tk.Misc.tkraise(report_canvas), explicitly raising the widget frame.
- Added regression coverage that previews screen_capture.png from the image-only fallback report path.
